SQL (Structured Query Language): a standard programming language used for managing and manipulating relational databases.

Relational database: is a type of database that organizes data into tables that are related to each other based on common attributes. Each table has a unique name, and in each table, each row is a record, while each column represents an attribute (a field).

Each column has a name and a data type. There are 7 kinds of data types:

  • Integer: Represents whole numbers without decimal points (e.g., INT, BIGINT).
  • Decimal and Float: Used for fractional numbers (e.g., 2.0, 3.69). The difference between them is that Decimal is used for precise decimal arithmetic, such as financial calculations and currency representation. Float is used for approximate arithmetic with real numbers.
  • Char and Varchar: Used for characters and strings. CHAR is for fixed-length strings, and VARCHAR is for variable-length strings.
  • Date and Time: Used for date and time. (e.g., DATE, DATETIME, TIMESTAMP)

SQL constraints are rules applied to columns in a table to ensure the accuracy and reliability of the data within the database.

  • PRIMARY KEY: Each table can have only one primary key, and the primary key column(s) cannot contain NULL values. It uniquely identifies each record in a table. 
    
      CREATE TABLE departments (
          department_id INT PRIMARY KEY,
          department_name VARCHAR(100)
      );

      CREATE TABLE employees (
          employee_id INT PRIMARY KEY,
          name VARCHAR(100),
          department_id INT,
          FOREIGN KEY (department_id) REFERENCES departments(department_id)
      );
  • FOREIGN KEY: Ensures that the value in one table matches a value in another table, helping maintain referential integrity. 
    
      CREATE TABLE departments (
          department_id INT PRIMARY KEY,
          department_name VARCHAR(100)
      );

      CREATE TABLE employees (
          employee_id INT PRIMARY KEY,
          name VARCHAR(100),
          department_id INT,
          FOREIGN KEY (department_id) REFERENCES departments(department_id)
      );
  • UNIQUE: Ensures that all values in a column are unique. A table can have multiple unique constraints. 
    
      CREATE TABLE users (
        user_id INT PRIMARY KEY,
        email VARCHAR(255) UNIQUE
      );
  • NOT NULL: Ensures that a column cannot have NULL values. 
    
      CREATE TABLE products (
        product_id INT PRIMARY KEY,
        product_name VARCHAR(100) NOT NULL
      );
  • DEFAULT: Provides a default value for a column when none is specified. 
    
      CREATE TABLE orders (
        order_id INT PRIMARY KEY,
        order_date DATE DEFAULT CURRENT_DATE
      );
  • CHECK: Ensures that the value in a column meets a specific condition. 
    
      CREATE TABLE employees (
          employee_id INT PRIMARY KEY,
          salary DECIMAL(10, 2),
          CHECK (salary > 0)
      );
  • AUTO_INCREMENT (MySQL-specific): Typically used for primary key columns, automatically generates a unique value for each new record. 
    
      CREATE TABLE employees (
          employee_id INT AUTO_INCREMENT PRIMARY KEY,
          name VARCHAR(100)
      );
